Solution

It appears that RSA has been used based on the values of N and e inside data.txt.

We are also given long integers within the wrapper gigem{}, which is the flag format, and based on this, we can assume that the long integers within the wrapper are ciphertexts.

As is typical in RSA challenges, let’s divide N into factors using factordb and determine the values for p and q. However, factordb does not provide a solution for the given value of N.

What else could we do?

Inside the wrapper, we can see that there are exactly 40 ciphertexts.

Could this imply that each character of the flag is encrypted separately? If that’s the case, we could write a python script for a dictionary attack against RSA.

When I was working on solving this challenge, I wrote the following script.

e = 65537
ciphertext = [int(i) for i in open("gigem.txt").read().split(" ")]

chars = string.printable
plaintext = [""] * len(ciphertext)

for char in chars:
    ct = pow(ord(char), e, N)

    for en, _ct in enumerate(ciphertext):
        if ct == _ct:
            plaintext[en] = char

print("gigem{" + "".join(plaintext) + "}")

Upon running the script, we get the flag gigem{enumerable_SeArCh_SpAcEs_4R3_WEAK_0xBEEF}.
